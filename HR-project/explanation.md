# HR Records System

## What it does
Manages structured employee records — storing, updating, and looking up staff data such as names, roles, or salaries.

## How it works
1. Store employee records in a dictionary (or list of dictionaries), keyed by employee ID or name.
2. Provide options to add a new record, update an existing one, or look one up.
3. Print the record in a readable format when requested.

## Why I built it
Practice structuring and manipulating data with dictionaries instead of just single variables — closer to how real data gets handled.

## Skills used
Dictionaries, functions, loops, user input, data lookup and formatting.

## Example
```
Add employee: Ahmed, Role: Technician, Salary: 8000
Lookup: Ahmed
Result: Role - Technician, Salary - 8000
```
